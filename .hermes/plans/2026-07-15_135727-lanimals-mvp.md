# LANimals MVP Implementation Plan

> **For Hermes:** Implement task-by-task using strict TDD and independently review the final diff.

**Goal:** Deliver a working cross-platform, source-run LAN group chat with password login, animal device identities, persistent text/file history, real-time updates, and host-only destructive controls.

**Architecture:** Python 3.11 FastAPI service with SQLite metadata, local upload storage, WebSocket broadcasts, and a framework-free responsive web UI. All browser clients are non-destructive; clearing data is only available through a local host CLI. Runtime configuration lives in ignored `data/config.toml`.

**Tech Stack:** Python 3.11, FastAPI, Uvicorn, SQLite, WebSocket, HTML/CSS/JavaScript, pytest.

---

## Confirmed requirements

- Product, package, and command name: `LANimals` / `lanimals`.
- Develop on Linux; run source primarily on Windows later.
- One password-protected shared room, no accounts.
- Persistent animal identity Cookie and session-only incognito identity Cookie.
- Text and arbitrary file messages persist across restarts.
- Browser UI/API has no delete capability.
- Host CLI alone can clear all messages and uploaded files.
- Host controls per-file upload size via `data/config.toml`.
- No external services, CDN, analytics, cloud storage, or public-internet dependency.

## Tasks

1. Rename the package from `app` to `lanimals`; add a CLI and configuration loader.
2. Harden authentication and sessions while preserving normal/incognito animal identity behavior.
3. Implement paginated message history and authenticated WebSocket broadcast.
4. Implement streamed file upload, configured size enforcement, safe local storage, and authenticated download.
5. Implement `python -m lanimals clear` with local confirmation and no browser deletion endpoint.
6. Build responsive login/chat UI with text, file progress, history, reconnection, and safe text rendering.
7. Add Linux/Windows source-run instructions, first-run setup, Git hygiene, and operational documentation.
8. Run the complete test suite, static checks, live HTTP/WebSocket smoke tests, and independent code review.

## Expected paths

- `lanimals/__main__.py`: `serve`, `clear`, and config commands.
- `lanimals/main.py`: FastAPI app factory and routes.
- `lanimals/config.py`: TOML configuration and first-run handling.
- `lanimals/identity.py`: cookie-backed devices and sessions.
- `lanimals/store.py`: SQLite persistence.
- `lanimals/realtime.py`: WebSocket hub.
- `lanimals/web/`: embedded static UI.
- `tests/`: behavioral and integration tests.
- `data/`: ignored runtime DB, config, and uploads.

## Verification

- Normal browser retains its animal; incognito Cookie is session-only.
- Two clients exchange text in real time and history survives restart.
- Files of allowed types/sizes upload and download; oversized/incomplete uploads leave no message or orphan file.
- No HTTP endpoint can delete messages/files.
- Host CLI clearing removes DB history and uploaded files after explicit confirmation.
- Server binds to LAN and the UI works from a second browser/client.
- `pytest -q` passes without new warnings caused by project code.

## Main risks

- Plain HTTP is only appropriate on a trusted LAN.
- SQLite and local storage are single-host by design.
- Browser request limits and disk exhaustion require defensive upload checks and free-space errors.
- Windows startup/firewall behavior needs final on-device verification even when Linux tests pass.
